# Phase 1 — Foundation: Admin Auth, Model Extensions, Registration Reset & Policy UX Fix

---

## Phase Goal

This phase establishes the critical infrastructure that every subsequent phase depends on. It resolves four foundational problems:

1. **Admin Panel Independent Authentication** — The dashboard at `/dashboard/admin/` must authenticate with Django username/password only. No Telegram auth required.
2. **TelegramUser + ServiceProfile Model Extensions** — Add GPS, last_interaction, likes_count, admin_forced_hidden, denial_count, penalty_until, rejection_reason fields via safe additive migrations.
3. **Registration Reset Button** — Every bot registration step must include a reset button that wipes session data but preserves policy acceptance.
4. **Policy Wrong-Answer Message Order Fix** — The policy text must be sent BEFORE the wrong-answer notification.
5. **Contact Limit Reconfiguration** — Adjust daily/weekly contact request limits and cooldown schedule.
6. **Unregistered Users as Customers** — Verify and document the existing correct customer-default behavior.

---

## Background

The existing system is a Telegram bot + Django REST API marketplace.

- Bot webhook: `api/bot/webhook/`
- Admin dashboard: `dashboard/admin/` protected by `@login_required` + `dashboard_user_can_access()`
- `TelegramUser` model stores all marketplace identities
- `BotRegistrationSession` stores multi-step provider registration state
- `bot/policy.py` manages policy text and quiz logic
- `approvals/usage_limits.py` manages daily contact request rate limiting
- Django built-in `auth.User` is used for admin dashboard login

## Detailed Requirements

### 1. Admin Dashboard Authentication Fix

#### Problem
`dashboard_user_can_access(request)` calls `get_dashboard_admin_telegram_user()` which requires a TelegramUser linked to the Django auth user. Django staff users without a linked TelegramUser are denied access.

#### Fix
Modify `dashboard_user_can_access(request)`:
- If `request.user.is_staff or request.user.is_superuser` is `True`, return `True` immediately. No TelegramUser lookup needed.

Modify `get_dashboard_admin_telegram_user(request)`:
- For staff/superuser with no linked TelegramUser, return `None` gracefully (do not raise).

Update `admin_dashboard` view:
- Must render even if `get_dashboard_admin_telegram_user()` returns `None`.
- Pass `admin_tg_user=None` to context. Template must handle this with `{% if admin_tg_user %}` guards.

Update `dashboard_contact_action` and `dashboard_service_action`:
- If `admin_user` is `None`, set `approved_by=None` on the record. Do not crash.
- Log: `logger.warning("event=admin_action_no_telegram_user user=%s action=%s", request.user.username, action)`

#### Behavior
- Django superuser navigates to `/dashboard/admin/` after standard Django login at `/admin/login/`.
- Dashboard renders without errors.
- Approve and reject work. `approved_by` is NULL in database when no TelegramUser exists.
- No redirect loop. No 403 for valid staff users.

#### Validation
- `@login_required` handles unauthenticated redirects (already present).
- Return 403 only if user is authenticated but is neither `is_staff`, `is_superuser`, nor has a TelegramUser with `role=ADMIN`.

### 2. TelegramUser Model Extensions

In `accounts/models.py`, add these fields to the `TelegramUser` class:

```
customer_latitude           DecimalField(max_digits=9, decimal_places=6, null=True, blank=True, help_text="Customer GPS latitude for proximity matching.")
customer_longitude          DecimalField(max_digits=9, decimal_places=6, null=True, blank=True, help_text="Customer GPS longitude for proximity matching.")
customer_location_updated_at  DateTimeField(null=True, blank=True, help_text="When customer last submitted their GPS location.")
last_interaction_at         DateTimeField(null=True, blank=True, db_index=True, help_text="Timestamp of user's last interaction with the bot or mini app.")
likes_count                 PositiveIntegerField(default=0, db_index=True, help_text="Total number of swipe-likes received by this provider.")
admin_tested_badge          BooleanField(default=False, db_index=True, help_text="Admin has personally tested and verified this provider.")
```

Update `TelegramUser.clean()`:
- If `customer_latitude` is set but `customer_longitude` is not (or vice versa), raise `ValidationError` requiring both.
- Validate `customer_latitude` is between -90 and 90.
- Validate `customer_longitude` is between -180 and 180.
- Also validate that the GPS is within Ethiopia's bounding box: latitude between 3.4 and 15.0, longitude between 33.0 and 48.0. If outside this range, do NOT raise an error during save — instead, set a flag or handle via a bot notification (see Phase 2 for the notification). For Phase 1, validation only logs a warning.

Add property:
```python
@property
def has_customer_location(self) -> bool:
    return self.customer_latitude is not None and self.customer_longitude is not None
```

Add method:
```python
def update_last_interaction(self, save: bool = True) -> None:
    self.last_interaction_at = timezone.now()
    if save:
        type(self).objects.filter(pk=self.pk).update(last_interaction_at=self.last_interaction_at)
```

Note: Use `objects.filter().update()` in `update_last_interaction` to avoid calling `full_clean()` on every interaction. This is a performance-critical path.

Add composite index on `(customer_latitude, customer_longitude)` in `Meta.indexes`.
Add index on `last_interaction_at` in `Meta.indexes`.

Migration: New file `accounts/migrations/XXXX_add_customer_location_and_interaction.py`. Use `migrations.AddField` for each field. Use `migrations.AddIndex` for new indexes.

```python
def handle_reset_callback(
    bot: TelegramBotService,
    context: TelegramUpdateContext,
) -> BotRouteResult:
    if context.chat_id is None or context.telegram_user_id is None:
        return BotRouteResult(False, "registration.reset.no_identity", context.chat_id, context.update_id)

    RegistrationStateMachine.start_or_reset(
        telegram_user_id=context.telegram_user_id,
        chat_id=context.chat_id,
        telegram_username=context.username or "",
    )

    bot.send_text(
        context.chat_id,
        "🔄 Registration reset. Starting from the beginning.\n\nPlease select your role:",
        reply_markup=bot.build_role_keyboard(),
    )

    return BotRouteResult(True, "registration.reset", context.chat_id, context.update_id)
```

Policy data (`policy_accepted_at`, `policy_version`) is on `TelegramUser`, NOT on `BotRegistrationSession`. `start_or_reset()` only touches the session, so policy data is preserved.

Replace `progressive_cooldown_seconds()` with lookup-table implementation:
```python
_COOLDOWN_TABLE_SECONDS = [0, 0, 1200, 1800, 3000, 3600]

def progressive_cooldown_seconds(requests_today: int) -> int:
    idx = min(requests_today, len(_COOLDOWN_TABLE_SECONDS) - 1)
```

Remove the old `BASE_CONTACT_COOLDOWN_SECONDS` and `MAX_CONTACT_COOLDOWN_SECONDS` constants.

Add helper `get_local_week_window(now)` function:
```python
def get_local_week_window(now):
    current_timezone = timezone.get_current_timezone()
    local_day = timezone.localdate(now, timezone=current_timezone)
    days_since_monday = local_day.weekday()  # 0=Monday
    week_start_date = local_day - timedelta(days=days_since_monday)
    week_start = timezone.make_aware(
        datetime.combine(week_start_date, time.min),
        current_timezone,
    )
    next_week_start = week_start + timedelta(days=7)
    return week_start, next_week_start
```

```python
requests_this_week = ContactRequest.objects.filter(

    created_at__gte=week_start,
    created_at__lt=next_week_start,
).count()

if requests_this_week >= WEEKLY_CONTACT_REQUEST_HARD_LIMIT:
    retry_after_seconds = seconds_until(now, next_week_start)
    return ContactRequestUsageDecision(
        allowed=False,
        requests_today=...,  # still compute this for completeness

        weekly_requests=requests_this_week,
        weekly_hard_limit=WEEKLY_CONTACT_REQUEST_HARD_LIMIT,
        protection_level="weekly_lock",
        cooldown_seconds=retry_after_seconds,
        retry_after_seconds=retry_after_seconds,
```
